57 """
58 iteratorsize(itertype::Type) -> IteratorSize
59
60 Given the type of an iterator, returns one of the following values:
61
62 * `SizeUnknown()` if the length (number of elements) cannot be determined in advance.
63 * `HasLength()` if there is a fixed, finite length.
64 * `HasShape()` if there is a known length plus a notion of multidimensional shape (as for an array).
65 In this case the [`size`](@ref) function is valid for the iterator.
66 * `IsInfinite()` if the iterator yields values forever.
67
68 The default value (for iterators that do not define this function) is `HasLength()`.
69 This means that most iterators are assumed to implement [`length`](@ref).
70
71 This trait is generally used to select between algorithms that pre-allocate space for their
72 result, and algorithms that resize their result incrementally.
73
74 ```jldoctest
75 julia> Base.iteratorsize(1:5)
76 Base.HasShape()
77
78 julia> Base.iteratorsize((2,3))
79 Base.HasLength()
80 ```
81 """
